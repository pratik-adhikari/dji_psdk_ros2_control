Transform from the configured frame (UTM or local) back to WGS84.

The input is (longitude, latitude) = (Easting, Northing) in UTM or local. The output is a WGS84 coordinate in (longitude, latitude) in degrees.

Configure transformations for UTM or LOCAL.

For FrameType::UTM, you can specify the zone and hemisphere if you're constructing a proj string manually. Alternatively, you can store an EPSG code for your zone to do a direct proj_create_crs_to_crs.

Parameters
frameTypeWhich output frame to use
zoneIf UTM, the zone (1..60)
northernIf UTM, whether in the northern hemisphere
baseLatIf LOCAL, the base latitude in degrees
baseLonIf LOCAL, the base longitude in degrees
baseAltIf LOCAL, the base altitude in meters

Transform from WGS84 (EPSG:4326) to the configured frame (UTM or local).

IMPORTANT: The input coordinate here is assumed to be in (longitude, latitude) for convenience. Internally, we will reorder it to (latitude, longitude) since EPSG:4326 in PROJ expects lat,lon. The output coordinate has (longitude, latitude) representing (Easting, Northing) in UTM, or local X, Y if FrameType::LOCAL.

Parameters
inWgsInput in degrees: (longitude, latitude, altitude)
outFrameOutput in the new frame (e.g. easting, northing, altitude)
